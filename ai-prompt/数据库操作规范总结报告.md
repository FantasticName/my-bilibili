# 数据库操作规范总结报告

> 本文档总结了 `xianyu-trading-platform` 项目中数据库操作的完整架构与编码规范，可直接复用到其他项目中。

---

## 一、整体架构概览

本项目采用 **无框架纯 JDBC** 方案操作数据库，核心由以下三层组成：

```
+-------------------------------------------------+
|              Service Impl 层                     |
|  使用 TxManager.executeInTransaction() 包裹      |
|  多步 DAO 调用，保证事务一致性                    |
+-------------------------------------------------+
|              DAO 层                              |
|  通过 ConnectionHolder.borrow() 获取连接         |
|  有事务上下文则复用，无则自行借还                 |
+-------------------------------------------------+
|         基础设施层（TxManager + DataSource）      |
|  TxManager 通过 ThreadLocal 绑定连接             |
|  DataSourceProvider 单例管理手写连接池           |
+-------------------------------------------------+
```

**核心设计思想**：Service 层负责开启事务边界，DAO 层感知事务上下文并复用连接。同一个请求线程内，多个 DAO 调用共享同一个 JDBC Connection，由 TxManager 统一负责 commit/rollback。

## 二、关键类源码与详细解析

### 2.1 SimpleConnectionPool -- 手写数据库连接池

**职责**：实现一个基于 `BlockingQueue` 的线程安全连接池，实现 `javax.sql.DataSource` 接口，支持连接的复用、有效性检查、超时保护。

```java
package io.github.fantasticname.xianyutradingplatform.util;

import org.slf4j.Logger;
import org.slf4j.LoggerFactory;

import javax.sql.DataSource;
import java.io.PrintWriter;
import java.sql.*;
import java.util.concurrent.ArrayBlockingQueue;
import java.util.concurrent.BlockingQueue;
import java.util.concurrent.TimeUnit;
import java.util.concurrent.atomic.AtomicInteger;
import java.util.logging.Logger;

/**
 * 手写简单数据库连接池 - 教学版
 * 
 * 核心设计思想：
 * 1. 连接复用：避免频繁创建/销毁物理连接，提升性能
 * 2. 线程安全：多线程环境下借还连接不出错
 * 3. 资源可控：通过maxPoolSize限制最大连接数，防止数据库被压垮
 * 4. 透明代理：外部代码感知不到"池"的存在，用法和普通Connection完全一致
 * 
 * 核心组件：
 * - idleConnections: BlockingQueue<PooledConnection>  存放空闲连接的阻塞队列
 * - totalConnections: AtomicInteger                    当前已创建的物理连接总数（含借出的）
 * - PooledConnection: 内部类，包装真实Connection的代理类，重写close()实现"归还而非关闭"
 * 
 * 借连接三步策略（getConnection）：
 * 1. 优先从空闲队列拿 -> 2. 队列空则尝试创建新连接（CAS防超限）-> 3. 池满则阻塞等待（带超时）
 */
public class SimpleConnectionPool implements DataSource {

    private static final Logger log = LoggerFactory.getLogger(SimpleConnectionPool.class);

    private final String jdbcUrl;
    private final String username;
    private final String password;
    private final int maxPoolSize;

    private final BlockingQueue<PooledConnection> idleConnections;
    private final AtomicInteger totalConnections = new AtomicInteger(0);
    private static final String VALIDATION_QUERY = "SELECT 1";

    private PrintWriter logWriter;
    private int loginTimeout;

    public SimpleConnectionPool(String jdbcUrl, String username, String password, int maxPoolSize) {
        this.jdbcUrl = jdbcUrl;
        this.username = username;
        this.password = password;
        this.maxPoolSize = maxPoolSize;
        this.idleConnections = new ArrayBlockingQueue<>(maxPoolSize);

        log.info("正在初始化手写连接池 | url={} | maxPoolSize={}", jdbcUrl, maxPoolSize);

        try {
            Class.forName("com.mysql.cj.jdbc.Driver");
            log.debug("MySQL JDBC Driver 注册成功");
        } catch (ClassNotFoundException e) {
            log.error("MySQL JDBC Driver 未找到！", e);
            throw new RuntimeException("MySQL JDBC Driver not found", e);
        }

        log.info("连接池初始化完成");
    }

    @Override
    public Connection getConnection() throws SQLException {
        return getConnection(username, password);
    }

    @Override
    public Connection getConnection(String username, String password) throws SQLException {
        // 第一步：尝试从空闲队列获取
        PooledConnection conn = idleConnections.poll();
        if (conn != null) {
            if (isConnectionValid(conn)) {
                return conn;
            } else {
                closePhysically(conn);
                totalConnections.decrementAndGet();
                return getConnection(username, password);
            }
        }

        // 第二步：队列为空，判断是否还能创建新连接（CAS原子操作防超限）
        int current = totalConnections.get();
        if (current < maxPoolSize) {
            if (totalConnections.compareAndSet(current, current + 1)) {
                try {
                    Connection physical = DriverManager.getConnection(jdbcUrl, username, password);
                    return new PooledConnection(physical, this);
                } catch (SQLException e) {
                    totalConnections.decrementAndGet();
                    throw e;
                }
            } else {
                return getConnection(username, password);
            }
        }

        // 第三步：池子已满，阻塞等待连接归还（带30秒超时保护）
        try {
            PooledConnection borrowed = idleConnections.poll(30, TimeUnit.SECONDS);
            if (borrowed == null) {
                throw new SQLException("Connection pool exhausted, timeout waiting for connection");
            }
            if (isConnectionValid(borrowed)) {
                return borrowed;
            } else {
                closePhysically(borrowed);
                totalConnections.decrementAndGet();
                return getConnection(username, password);
            }
        } catch (InterruptedException e) {
            Thread.currentThread().interrupt();
            throw new SQLException("Interrupted while waiting for connection", e);
        }
    }

    void returnConnection(PooledConnection conn) {
        if (conn == null) return;

        if (isConnectionValid(conn)) {
            boolean offered = idleConnections.offer(conn);
            if (!offered) {
                closePhysically(conn);
                totalConnections.decrementAndGet();
            }
        } else {
            closePhysically(conn);
            totalConnections.decrementAndGet();
        }
    }

    private boolean isConnectionValid(PooledConnection conn) {
        try {
            if (conn.isClosed()) {
                return false;
            }
            try (Statement stmt = conn.physicalConnection.createStatement()) {
                stmt.setQueryTimeout(3);
                stmt.execute(VALIDATION_QUERY);
            }
            return true;
        } catch (SQLException e) {
            return false;
        }
    }

    private void closePhysically(PooledConnection conn) {
        if (conn == null) return;
        try {
            if (!conn.physicalConnection.isClosed()) {
                conn.physicalConnection.close();
            }
        } catch (SQLException e) {
            log.warn("物理关闭连接时发生异常（已忽略）");
        }
    }

    // ... JDBC规范方法（getLogWriter/setLogWriter等）...

    /**
     * 核心设计：PooledConnection - Connection的代理类
     * 让外部代码以为自己在操作普通Connection，但实际调用close()时
     * 不是真的关闭数据库连接，而是"归还到连接池"，实现连接复用
     */
    static class PooledConnection implements Connection {

        private final Connection physicalConnection;
        private final SimpleConnectionPool pool;
        private boolean closed = false;

        PooledConnection(Connection physical, SimpleConnectionPool pool) {
            this.physicalConnection = physical;
            this.pool = pool;
        }

        /**
         * 关键重写：close() 方法
         * 普通Connection.close() -> 物理关闭数据库连接
         * PooledConnection.close() -> 归还连接到空闲队列
         */
        @Override
        public void close() throws SQLException {
            if (closed) {
                return;
            }
            closed = true;
            pool.returnConnection(this);
        }

        @Override
        public boolean isClosed() throws SQLException {
            return closed || physicalConnection.isClosed();
        }

        // 其他所有方法：直接委托给physicalConnection
        @Override
        public PreparedStatement prepareStatement(String sql) throws SQLException {
            checkClosed();
            return physicalConnection.prepareStatement(sql);
        }

        @Override
        public void setAutoCommit(boolean autoCommit) throws SQLException {
            checkClosed();
            physicalConnection.setAutoCommit(autoCommit);
        }

        @Override
        public void commit() throws SQLException {
            checkClosed();
            physicalConnection.commit();
        }

        @Override
        public void rollback() throws SQLException {
            checkClosed();
            physicalConnection.rollback();
        }

        // ... 其余JDBC Connection接口方法 ...

        private void checkClosed() throws SQLException {
            if (closed) {
                throw new SQLException("Connection is logically closed (returned to pool)");
            }
        }
    }
}
```

**要点**：
- 使用 `ArrayBlockingQueue` 作为空闲连接队列，天然线程安全
- 借连接采用三步策略：空闲队列 -> CAS创建新连接 -> 阻塞等待（30秒超时）
- `PooledConnection` 重写 `close()` 方法，实现"归还而非关闭"
- 每次获取连接前执行 `SELECT 1` 检查有效性，防止"僵尸连接"
- `AtomicInteger` + CAS 保证多线程环境下连接计数准确，防止超限

### 2.3 TxSupplier -- 事务回调接口

**职责**：定义一段"要在事务里跑"的业务逻辑。使用自定义接口而非 `java.util.function.Supplier` 的原因：
1. **见名知意**：看到 `TxSupplier` 就知道这段代码在事务上下文中执行
2. **预留扩展口**：未来如需在 `get()` 中注入上下文参数（如当前用户 ID），可直接修改接口签名

```java
package io.github.fantasticname.xianyutradingplatform.util;

@FunctionalInterface
public interface TxSupplier<T> {
    /**
     * 获取事务内执行结果。
     * 实现方通常在方法内部串联多个 DAO 调用；
     * 事务提交/回滚由 TxManager 负责。
     */
    T get();
}
```

### 2.4 TxManager -- 事务管理器（核心）

**职责**：为同一线程内的一组 DAO 操作提供一致的事务边界。

```java
package io.github.fantasticname.xianyutradingplatform.util;

import io.github.fantasticname.xianyutradingplatform.common.ErrorCode;
import io.github.fantasticname.xianyutradingplatform.exception.BusinessException;

import javax.sql.DataSource;
import java.sql.Connection;

public final class TxManager {
    // ThreadLocal 绑定当前线程的事务连接
    private static final ThreadLocal<Connection> CTX = new ThreadLocal<>();

    private TxManager() {}

    /**
     * 获取当前线程绑定的事务连接。
     * DAO 层通过此方法判断是否处于事务范围内：
     * - 返回非空 -> 复用该连接，提交/回滚由 TxManager 负责
     * - 返回 null  -> DAO 自行获取连接并独立关闭
     */
    public static Connection currentConnection() {
        return CTX.get();
    }

    /**
     * 在一个数据库事务中执行给定的业务逻辑。
     */
    public static <T> T executeInTransaction(DataSource ds, TxSupplier<T> supplier) {
        // 1. 检查是否已有外层事务连接
        Connection existing = CTX.get();
        if (existing != null) {
            // 已有连接 -> 直接复用，不开新事务
            return supplier.get();
        }

        // 2. 无外层事务 -> 自己开连接、开事务
        try (Connection conn = ds.getConnection()) {
            conn.setAutoCommit(false);  // 开启手动提交（即事务）
            CTX.set(conn);              // 绑定到当前线程

            try {
                T result = supplier.get();
                conn.commit();          // 成功 -> 提交
                return result;
            } catch (BusinessException e) {
                conn.rollback();        // 业务异常 -> 回滚并透传
                throw e;
            } catch (Exception e) {
                conn.rollback();        // 非业务异常 -> 回滚并包装
                throw new BusinessException(ErrorCode.INTERNAL_ERROR, "事务执行失败");
            } finally {
                CTX.remove();           // 必须清理 ThreadLocal，防止连接泄漏
                conn.setAutoCommit(true); // 恢复自动提交，连接可回池复用
            }
        } catch (BusinessException e) {
            throw e;
        } catch (Exception e) {
            throw new BusinessException(ErrorCode.INTERNAL_ERROR, "数据库连接失败");
        }
    }
}
```

**事务执行流程图**：

```
Service 调用 TxManager.executeInTransaction(ds, () -> {
    |
    +-- 检查 CTX.get() -> null（无外层事务）
    |   +-- ds.getConnection() -> 获取物理连接 conn
    |   +-- conn.setAutoCommit(false) -> 开启事务
    |   +-- CTX.set(conn) -> 绑定到当前线程
    |   |
    |   +-- supplier.get() 执行中...
    |   |   +-- DAO A 调用 TxManager.currentConnection() -> 拿到 conn，复用
    |   |   +-- DAO B 调用 TxManager.currentConnection() -> 拿到 conn，复用
    |   |   +-- DAO C 调用 TxManager.currentConnection() -> 拿到 conn，复用
    |   |
    |   +-- 无异常 -> conn.commit()
    |   +-- 有 BusinessException -> conn.rollback() + 透传
    |   +-- 有其他 Exception -> conn.rollback() + 包装为 BusinessException
    |
    +-- finally:
        +-- CTX.remove() -> 解除线程绑定
        +-- conn.setAutoCommit(true) -> 恢复状态
        +-- try-with-resources -> conn.close() -> 归还连接池
```

### 2.5 DAO 层的 ConnectionHolder 模式

每个 DAO 都包含一个内部类 `ConnectionHolder` 和 `borrow()` 方法，用于智能获取连接：

```java
// 每个 DAO 中重复出现的模式：

private ConnectionHolder borrow() {
    Connection tx = TxManager.currentConnection();
    if (tx != null) {
        // 在事务中 -> 复用事务连接，标记为"非借用"
        return new ConnectionHolder(tx, false);
    }
    // 不在事务中 -> 自己从连接池借一个，标记为"借用"
    try {
        return new ConnectionHolder(ds.getConnection(), true);
    } catch (Exception e) {
        throw new IllegalStateException(e);
    }
}

private static final class ConnectionHolder {
    private final Connection conn;
    private final boolean borrowed;  // true = 自己借的，需要自己关

    private ConnectionHolder(Connection conn, boolean borrowed) {
        this.conn = conn;
        this.borrowed = borrowed;
    }

    public Connection conn() {
        return conn;
    }

    public void closeIfBorrowed() {
        if (!borrowed) {
            return;  // 事务连接，不关！交给 TxManager 处理
        }
        try {
            conn.close();  // 自己借的，自己关（归还连接池）
        } catch (Exception ignored) {
        }
    }
}
```

**核心逻辑**：
- `borrowed = false`：连接来自 TxManager 的事务上下文，DAO **不关闭**，由 TxManager 统一 commit/rollback/close
- `borrowed = true`：连接是 DAO 自己从连接池借的，DAO 在 finally 中**必须关闭**（归还连接池）

```java
public List<Xxx> search(String keyword, String category, int offset, int pageSize) {
    StringBuilder sb = new StringBuilder();
    List<Object> params = new ArrayList<>();
    sb.append("SELECT id, col1, col2 FROM xxx_table WHERE 1=1");

    if (keyword != null && !keyword.isBlank()) {
        sb.append(" AND col1 LIKE ?");
        params.add("%" + keyword.trim() + "%");
    }
    if (category != null && !category.isBlank()) {
        sb.append(" AND category = ?");
        params.add(category.trim());
    }
    sb.append(" ORDER BY created_at DESC LIMIT ? OFFSET ?");
    params.add(pageSize);
    params.add(offset);

    ConnectionHolder ch = borrow();
    Connection conn = ch.conn();
    try (PreparedStatement ps = conn.prepareStatement(sb.toString())) {
        bind(ps, params);
        List<Xxx> out = new ArrayList<>();
        try (ResultSet rs = ps.executeQuery()) {
            while (rs.next()) {
                out.add(map(rs));
            }
        }
        return out;
    } catch (Exception e) {
        throw new IllegalStateException(e);
    } finally {
        ch.closeIfBorrowed();
    }
}

private void bind(PreparedStatement ps, List<Object> params) throws Exception {
    for (int i = 0; i < params.size(); i++) {
        Object v = params.get(i);
        int idx = i + 1;
        if (v instanceof String s) {
            ps.setString(idx, s);
        } else if (v instanceof Integer n) {
            ps.setInt(idx, n);
        } else {
            ps.setObject(idx, v);
        }
    }
}
```

#### 3.3.2 需要事务的方法（多步 DAO 操作）

**规则**：只要一个方法内调用了 **2 个或以上** 的写操作 DAO 方法，或者先读后写需要保证一致性，就必须用 `TxManager.executeInTransaction()` 包裹。

```java
@Override
public XxxVO create(String userId, CreateXxxRequest request) {
    // 1. 参数校验（在事务外做）
    validateCreate(request);

    // 2. 事务包裹
    return TxManager.executeInTransaction(ds, () -> {
        // 3. 构建领域对象
        Xxx x = new Xxx();
        x.setId(UuidUtil.newUuid());
        x.setCreatedAt(LocalDateTime.now());
        // ... 设置其他字段 ...

        // 4. 执行 DAO 写操作
        xxxDao.insert(x);

        // 5. 可选：查询关联数据组装 VO
        User user = userDao.findById(userId);
        return toXxxVO(x, user);
    });
}
```

#### 3.3.3 不需要事务的方法（单步读操作）

**规则**：只调用一个 DAO 的查询方法，不需要事务包裹。

#### 3.3.4 删除操作（需要事务 + 权限校验）

```java
@Override
public void delete(String userId, String id) {
    TxManager.executeInTransaction(ds, () -> {
        Xxx existed = xxxDao.findById(id);
        if (existed == null) {
            throw new BusinessException(ErrorCode.NOT_FOUND, "记录不存在");
        }
        if (!userId.equals(existed.getOwnerId())) {
            throw new BusinessException(ErrorCode.FORBIDDEN, "只能删除自己的记录");
        }
        xxxDao.deleteById(id);
        return null;
    });
}
```

## 四、编码规范速查表

| 层级 | 规则 | 说明 |
|------|------|------|
| **SimpleConnectionPool** | 基于 `BlockingQueue` + CAS 实现线程安全 | 空闲队列 + 原子计数 + 超时等待 |
| **SimpleConnectionPool** | `PooledConnection.close()` 是归还不是关闭 | 外部代码无感知，用法和普通 Connection 一致 |
| **SimpleConnectionPool** | 每次取连接前执行 `SELECT 1` 检查有效性 | 防止"僵尸连接"导致 SQL 执行失败 |
| **SimpleConnectionPool** | 池满时阻塞等待 30 秒，超时抛异常 | 防止无限等待，暴露连接泄漏问题 |
| **DataSourceProvider** | 单例模式，饿汉式初始化 | 应用启动时创建一次，全局复用 |
| **TxManager** | ThreadLocal 绑定连接 | 保证同一线程内 DAO 共享同一连接 |
| **TxManager** | `CTX.remove()` 必须在 finally 中调用 | 防止连接泄漏到后续请求 |
| **TxManager** | 检测到已有连接时直接复用 | 不支持嵌套事务，避免重复 commit |
| **DAO** | 每个 DAO 必须包含 `borrow()` + `ConnectionHolder` | 智能判断是否在事务中 |
| **DAO** | `ch.closeIfBorrowed()` 必须在 finally 中调用 | 确保非事务连接被正确归还 |
| **DAO** | 使用 `PreparedStatement` + `?` 占位符 | 防止 SQL 注入 |
| **DAO** | 异常统一包装为 `IllegalStateException` | DAO 不抛业务异常 |
| **DAO** | 只做数据访问，不做业务校验 | 校验逻辑在 Service 层 |
| **Service** | 多步写操作必须用 `executeInTransaction()` 包裹 | 保证原子性 |
| **Service** | 单步读操作不需要事务 | 减少不必要的连接占用 |
| **Service** | 业务异常抛 `BusinessException` | TxManager 会回滚并透传 |
| **Service** | 参数校验在事务外做 | 避免无效请求占用连接 |

## 六、注意事项

1. **ThreadLocal 清理**：`TxManager` 的 `finally` 块中 `CTX.remove()` 是必须的。本项目基于 Jetty/Tomcat 线程池，请求结束后线程不会销毁，如果不清理 ThreadLocal，连接会泄漏到下一个请求。

2. **自动提交恢复**：`conn.setAutoCommit(true)` 在 finally 中恢复，确保连接归还连接池时处于干净状态。

3. **DAO 独立调用**：DAO 方法可以在事务外单独调用（此时 DAO 自己借连接、自己关），也可以在事务内调用（复用事务连接）。这是 `ConnectionHolder` 模式的核心价值。

4. **异常策略**：
   - DAO 层：捕获所有异常，包装为 `IllegalStateException` 抛出
   - Service 层：业务校验失败抛 `BusinessException`
   - TxManager：`BusinessException` 透传 + 回滚；其他异常包装为 `BusinessException(INTERNAL_ERROR)` + 回滚

5. **枚举存储**：Java 枚举通过 `.name()` 转为字符串存入数据库，读取时通过 `Enum.valueOf()` 还原。

6. **时间处理**：Java 端使用 `LocalDateTime`，数据库端使用 `TIMESTAMP`，通过 `Timestamp.valueOf()` 和 `Timestamp.toLocalDateTime()` 转换。

7. **连接池有效性检查**：`SimpleConnectionPool` 每次从空闲队列取出连接时会执行 `SELECT 1` 验证连接是否仍然有效。如果连接已失效（如网络断开、数据库重启），会物理销毁该连接并重新获取，防止"僵尸连接"导致 SQL 执行失败。

8. **连接池超时保护**：当连接池已满（所有连接都在使用中），后续请求会阻塞等待最多 30 秒。如果 30 秒内仍未获取到可用连接，会抛出 `SQLException`。这说明系统负载过高或存在连接泄漏，需要排查。

9. **CAS 原子操作防超限**：`SimpleConnectionPool` 使用 `AtomicInteger.compareAndSet()` 保证多线程环境下创建连接数不会超过 `maxPoolSize`。即使多个线程同时判断"还能创建"，也只有一个线程能成功 CAS，其他线程会递归重试。

10. **PooledConnection.close() 是归还不是关闭**：手写连接池的核心在于 `PooledConnection` 重写了 `close()` 方法，使其行为从"物理关闭数据库连接"变为"归还到空闲队列"。外部代码使用 `try-with-resources` 时会自动调用此方法，确保连接被正确复用。
