  ```java
  import lombok.Data;

  /**
   * 通用返回结果类.实现序列化接口，意味着这个类的对象可以被转换成字节流（用于网络传输、缓存到 Redis 等）
   * 
   * @param <T> 数据类型
   * @author FantasticName
   */
  @Data
  public class Result<T> implements Serializable {
  
      private int code;
  
      private T data;
  
      private String message;
  
      private String traceId;
  
      public Result(int code, T data, String message) {
          this.code = code;
          this.data = data;
          this.message = message;
      }
  
      public Result(int code, T data) {
          this(code, data, "");
      }
  
      public Result(ErrorCode errorCode) {
          this(errorCode.getCode(), null, errorCode.getMessage());
      }
  
  
      
      // 静态工厂方法如下
  
  
      /**
       * 成功
       *
       * @param data 数据
       * @param <T>  数据类型
       * @return Result

      public static <T> Result<T> success(T data) {
          return new Result<>(0, data, "ok");
      }
  
      /**
       * 失败
       *
       * @param errorCode 错误码
       * @return Result
  ```

- 关于日志:

  - 使用SLF4J + logback

  - 每一个类都必须打上详细的日志 , 不得偷懒

  - 日志配置:

    - 日志**同时输出到控制台 + 文件 . 日志文件统一放在`backend/logs`目录里**

      控制台和文件日志**格式一样、不乱码**

      日志**按天自动生成新文件**

      **只保留 30 天日志**

      **全局默认只打印 INFO 级别**

      **可以看情况在指定包下开启 DEBUG 更详细日志 , 现在暂时设置为在 `io.github.fantasticname.mybilibili`包下记录DEBUG级别日志**

    - 日志一定要有TraceId用来全链路追踪一个请求 , 不仅要使用日志框架现有的`MDC`, 还要自己手写一个工具类`TraceIdContext`用于把TraceId和当前线程绑定在一起 . 当请求进来的时候生成TraceId , 和当前线程绑在一起 , 后端返回响应对象`Result`的时候也要把TraceId塞进去

- TraceId 和 全局异常处理 可以依靠Filter chain实现

- 注释 

  - 按照阿里巴巴编码规范写javadoc注释 , author写FantasticName


  - **<u>在方法内, 也要使用单行注释 , 解释代码一步步是如何运行的 ( 重要 ! ! ! )</u>** . 单行注释尽量不要和代码同行 .

    示例如下:

    ````java
    @PostMapping("/upload")
    public Map<String, Object> uploadImage(@RequestParam Long orderId, @RequestParam MultipartFile file) {
        // 1. 调用 FileUtil 把图片存到硬盘，拿到相对路径（如 "uploads/xxx.jpg"）

        // 3. 如果数据库更新失败，就把刚才存的图片删掉，保证数据一致性
        if (!success) {
            FileUtil.deleteFile(imagePath);
        }
    
       
        return result;  
    }
    ````
