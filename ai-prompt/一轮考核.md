### 📖 项目背景

我要当赛博搬石大王

在互联网鉴石无数的你已经不再满足于自己赤石，你决定把遇到的石都收集起来，鼓励其他人和你一起收集奇石，欣赏构石，集齐五色石，补全破碎的虚假之天（参考文献：《日月前事》）

你相信只要你努力完善你的平台，你的赛博聚石盆一定能战胜 pilipala、踢踏、慢脚这种垃圾平台，让赤石的乐趣重新盛行

**目标：**

编写一个视频推流系统，主要让用户去观看博主发的有趣视频，给身边的人带来欢乐。

------

### 📋 技术要求

1. 架构：前后端分离，后端采用单体架构，并且要遵循 mvc 结构进行分层编写。
2. 后端语言：必须使用 Java。
3. 数据库：采用 MySQL。
4. 缓存：考虑到大家的学习进度不太一致，我们允许在一轮阶段用 Java 集合类来实现缓存，有能力可以采用 Redis 实现缓存。
5. 前端：不强制要求，有时间就弄一个简单的页面，没时间就算了，允许直接用控制台，不要用 jsp 和 swing！不要用 jsp 和 swing！不要用 jsp 和 swing！
6. 禁止：不允许使用 SpringBoot、SpringMVC 和 MyBatis 框架。如需使用其他框架，需要提前申请确认。我们鼓励手写框架，可以使用 Maven 进行管理开发。
7. 知识储备：JavaWeb 和 MySQL 的相关知识。

------

### 📌 业务需求

#### 功能模块：

1. **注册功能（必做）**

   

   - 实现用户注册和管理员注册功能
   - 将用户信息持久化到数据库中
   - 不要将明文密码存储到数据库（例如采用哈希加盐加密）

   

2. **登入功能（必做）**

   

   - 用户通过账号密码登入
   - 登入后生成对应 Token
   - 实现 Token 的刷新机制
   - 校验用户是否登入，并缓存用户的登入状态

   

3. **视频管理（必做）**

   

   - 支持查看视频
   - 缓存视频信息，优化读取性能

   

4. **评论功能（必做）**

   

   - 用户可以在视频下方进行评论和删除评论

   

5. **关注功能（进阶）**

   

   - 用户可以关注博主，也可以关注其他用户
   - 可以查询用户的关注列表
   - 可以查询博主的粉丝数量

   

6. **点赞功能（进阶）**

   

   - 可以对视频和评论进行点赞
   - 实现视频和评论的点赞数统计
   - 实现按照热度或者时间顺序来展示评论

   

7. **RBAC（进阶）**

   

   - 为每一个接口分配一个权限，用以限制用户对于管理员接口的访问（防止垂直越权）
   - 为每一个角色分配一组权限

   

------

### 🛠️ 技术要求

1. **MySQL 数据库连接池（必做）**

   

   - 自己编写一个简单的数据库连接池

   

2. **封装所编写的 crud 语句（必做）**

   

   - 借鉴 ORM 框架的一些思路，实现对数据库的增删改查进行一个基础的封装

   

3. **日志统一处理（必做）**

   

   - 在必要的地方（如数据库连接、发生异常、数据异常、事务失败等）统一记录日志（支持记录不同日志等级，使用 JDK 自带的日志类，位于`java.util.logging`包）
   - 可以了解一下面向切面编程（AOP）和动态代理（CGLIB）

   

4. **统一的 Bean 对象管理（进阶）**

   

   - 实现一个简单的 IoC 容器，实现全局的 Bean 对象管理

   

5. **统一异常处理（进阶）**

   

   - 对系统中不同的异常集中到一起处理

   

------

### 📁 代码提交要求

1. 代码结构：编写完整的 Java 项目，代码目录清晰，遵循项目结构规范。编写 README 文件说明如何运行项目及依赖环境。
2. 数据库设计：保留 SQL 建表语句，并且要注释字段含义。

4. 代码规范：遵循 Java 编程规范，代码清晰、注释完整。
